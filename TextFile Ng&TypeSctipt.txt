there 2 ways : 
1)using snapshot: Snapshot gives the current state of URL at the time component loads.
If value changes after component loaded, snapshot won’t update.  
ex: const id = this.route.snapshot.paramMap.get('id');

2)using observables:Route parameters are Observables because they can change while component is active (user clicks inside app).
So we subscribe to get values live whenever route changes.
ex:
this.route.params.subscribe((params) => {
  console.log(params['id']);
});

| Subscribe | Shopkeeper updates price & **keeps informing** continuously                                          |

// Signals: basically they are used for change detection. it works as a observables

 //understand why we use signals:
  //without signals:
  a = 2;
  b = 3;
  c = this.a + this.b;

  withoutSignalEx() {
    console.log('C before value of a change: ', this.c);
    this.a = 10;
    console.log('C after value of a change: ', this.c);
  }

  //with signal
  A1 = signal(10);
  B1 = signal(20);
  C1 = computed(() => this.A1() + this.B1()); //"computed" is Readonly signal type swhich contains the callback(arrow fn)

  withSignalEx() {
    console.log('C1 before value of A1 change: ', this.C1());
    this.A1.set(100);
    console.log('C1 after value of A1 change: ', this.C1());   ====> Here value will changed and bcz change in A1 reflects bcz of signal
  }

ngOnInit(){
withoutSignalEx();
withSignalEx();
}

==> signal ma pachhal thi value change krvi hoi to
